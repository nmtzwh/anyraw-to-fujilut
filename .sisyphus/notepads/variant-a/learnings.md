# Learnings
- Implementing a small CPU-based 3D LUT pipeline is feasible with pure NumPy.
- Use deterministic test data with small shapes to verify each step end-to-end.
- 3D LUT can be implemented via trilinear interpolation across a (N,N,N,3) grid.
- FastAPI TestClient is excellent for integration tests without needing a running server.
- multipart/form-data uploads need explicit file tuples: (filename, content, mime_type).
- Torch-free backend verified: zero torch/PyQt5 imports in backend/.
- Wave 1.3: backend/requirements.txt updated with explicit pins (numpy>=1.26,<2, scipy>=1.10, rawpy>=0.19, colour-science>=0.19, etc.). Torch absent; root requirements.txt still has torch (separate).
- Wave 1.4: 12 tests total (6 pipeline unit + 6 API/integration); new additions: out-of-range flog2 clamping test, end-to-end synthetic pipeline test, conftest.py sys.path setup, tests/README.md.
- Wave 2A.1: Electron scaffold created in .worktrees/electron-wave2a/ — main.ts (65 brace pairs, 4 imports), preload.ts (17 pairs), ipc.ts (44 pairs), public/index.html. electron module LSP errors are env-only (npm install not run yet). Torch-free confirmed in src/electron/. Security: contextIsolation:true, nodeIntegration:false, sandbox:true.
- Wave 2A.2: tsconfig split into tsconfig.base.json + tsconfig.main.json + tsconfig.renderer.json; package.json has separate build:main (CJS, node), build:renderer (IIFE, browser), and typecheck commands. Renderer stubs created (app.ts, color.ts, lut.ts, types.d.ts) so builds pass without full UI. preload.ts stays in node bundle (uses Electron APIs — cannot be browser-platform bundled).
- Wave 2A.3: src/app.ts fully rewritten with Electron dialogs (api.dialog.openFile/saveFile), backend IPC (api.backend.health/convert), fs bridge (api.fs.readFile/writeFile). Architecture shift: RAW decoding moved entirely to Python backend; renderer only handles UI state and IPC. Added fs:readFile to ipc.ts (needed because dialog returns path string, renderer needs bytes). qe<T> generic DOM helper avoids jQuery false-positive. All 6 source files pass brace-balance check.
- Wave 1.5 (pipeline integration): Full pipeline wired into /convert endpoint. Chain: load_image_to_xyz (rawpy/tifffile/imageio → XYZ) → xyz_to_rec2020 → apply_flog2_curve → apply_lut → float32→uint8→JPEG→base64. New modules: cube_parser.py (parses .cube files into (N,N,N,3) float32), raw_loader.py (RAW/TIFF/JPEG→XYZ). Temp file extension preserved from original filename via tempfile.mktemp(suffix=orig_ext) so the loader dispatches correctly. pipeline.prewarm() called in startup_event. test_main.py updated to use real TIFF bytes (not JPEG content with .tiff filename). All 25 tests pass across 5 test files.
- Wave 3 (E2E + CI + error modal): Added backend:simulateError IPC channel in ipc.ts (module-level flag, set via ipcMain.handle). Exposed in preload.ts and types.d.ts. Error modal added to public/index.html (fixed overlay with flex centering) and wired in app.ts: showErrorModal() called on convert catch, retry calls startConversion() again, dismiss closes modal. __showTestErrorModal test hook exposed on window for Playwright tests. Playwright config (playwright.config.ts) uses Electron launcher via browserName:'electron'. Tests/e2e/app.spec.ts: 9 tests covering health indicator, version, button state, modal show/hide/retry, simulateError injection. Test files use // @ts-nocheck (Playwright/@playwright/test not installed locally; resolved in CI). CI: e2e-tests job added to build.yml with matrix (ubuntu + windows), depends on lint-and-test. npm ci in CI installs @playwright/test and npx playwright install --with-deps chromium runs the browsers.
